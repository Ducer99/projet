Voici plusieurs modèles de messages que vous pouvez copier-coller et personnaliser.

---

🔗 [URL_DE_TEST]

## 📧 Version Email complète

**Objet** : Aide-moi à tester mon application d'arbre généalogique 🌳

```
Bonjour [Prénom],

J'ai développé une application web pour gérer un arbre généalogique familial 
et j'aurais besoin de ton retour pour l'améliorer ! 

🌐 **Lien de test** : [VOTRE_URL_ICI]

📄 **Guide de test** : [Lien vers le guide]

⏱️ **Temps estimé** : 15-30 minutes

---

## 🎯 Ce que je teste

Cette application permet de :
✅ Créer un compte et gérer son profil
✅ Créer ou rejoindre un arbre généalogique familial
✅ Ajouter des membres (parents, enfants, conjoints)
✅ Visualiser l'arbre de façon interactive
✅ Gérer les événements et mariages familiaux

---

## 🧪 Ce dont j'ai besoin

1. **Bugs techniques** : Erreurs, pages qui ne chargent pas, fonctionnalités cassées
2. **Expérience utilisateur** : Est-ce intuitif ? Facile à utiliser ?
3. **Design** : L'interface est-elle agréable ? Claire ?
4. **Suggestions** : Quelles améliorations proposerais-tu ?

---

## 📝 Comment me faire un retour

**Option 1** : Réponds directement à cet email avec tes commentaires

**Option 2** : Appelle-moi pour en discuter : [Ton numéro]

**Option 3** : Remplis ce formulaire : [Lien si tu en as un]

---

## 💡 Conseils pour tester

- Essaie de créer un compte avec ton vrai email
- N'hésite pas à cliquer partout pour explorer
- Note tout ce qui te semble bizarre ou compliqué
- Si tu peux, teste sur ton téléphone aussi !

---

## 🙏 Merci d'avance !

Ton aide est vraiment précieuse pour rendre cette application meilleure.
Même si tu ne trouves qu'un seul petit bug, ça m'aide énormément !

À très bientôt,
[Ton nom]

P.S. : Si tu connais d'autres personnes qui pourraient tester, n'hésite pas 
à partager le lien ! Plus on est nombreux, mieux c'est 😊
```

## 🎯 Mini-checklist avant d'envoyer

Avant d'envoyer votre message, vérifiez :

- [ ] **URL de test** : Remplacez `[VOTRE_URL_ICI]` par la vraie URL
- [ ] **Guide de test** : Ajoutez le lien vers le fichier GUIDE_TEST_UTILISATEURS.md
- [ ] **Contact** : Ajoutez vos vrais coordonnées (email, tél, etc.)
- [ ] **Date limite** : Indiquez une deadline pour les retours
- [ ] **Application testée** : Assurez-vous qu'elle fonctionne avant d'envoyer !
- [ ] **CORS configuré** : L'URL publique est bien autorisée dans le backend
- [ ] **Code d'invitation** : Si besoin, préparez des codes pour vos testeurs
